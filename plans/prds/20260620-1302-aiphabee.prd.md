## AI Quick-Read Card

- **Problem:** 个人投资者与其 AI Agent 缺少长期、结构化、可验证、可复用的港股研究能力；数据分散、口径不一致、数字无法追溯。
- **Users:** 自主研究型个人投资者、港股新手、资深投资者、AI Power User/开发者（P0）；内容创作者、研究团队、内部运营（P1/P2）。
- **Platform:** Web 研究 Agent + Remote MCP 数据/分析平台；TanStack Start + Vite + Hono + Vercel AI SDK v7 on Cloudflare Workers；Postgres/Supabase 主真值源。
- **P0 surface:** 自然语言研究、个股工作台、行情/财务趋势、2–5 股比较、公告检索摘要、筛选器、保存研究、来源证据、订阅配额；MCP 12+ 只读原子工具。
- **Core metric:** WVRO（周度已验证研究成果，PRD §4.3）。
- **Hard constraint:** Gate 0 数据再分发权 + Type 4 合规分类先于大规模产品代码；证据优先于语言；P0 工具全部只读；Web 与 MCP 权利分离；金融数字必须来自工具或确定性计算。
- **Key risk:** 数据合作方无 MCP/API 再分发权 → MCP 产品线无法上线（头号阻断）。
- **Unknowns:** 实时/延迟/EOD 首发口径；30 年历史是否全量经 MCP；Type 4 具体判定；见 PRD §21 全表（11 项待确认）。
- **Acceptance scenarios:** 见下「Acceptance Scenarios」与 PRD §17（US-W01..10 / US-M01..08 / US-O01..06）。
- **Suggested next step:** 执行 `plans/sprints/20260620-1307-phase0-gate0-foundation.sprint.md` 的 Gate 0 backlog；Gate 0 退出门槛全绿前不进入 Phase 1 特性开发。

## Problem

### Product Direction

- **Hard Constraints:**
  - Gate 0：字段级再分发权（Web/MCP/API/导出/派生）+ Type 4/研究工具分类书面意见先于大规模产品代码（PRD §0.2 决策 7、§14）。
  - 一个共享金融 Tool Registry，两个产品入口（Web Agent + MCP）共用工具、指标定义、权限规则与证据服务（§0.2 决策 1）。
  - Web 展示权与 MCP/API 再分发权分别确认、分别计费、分别管控；默认拒绝未授权字段（§0.2 决策 2、§14.1）。
  - 证据优先：每个数字带 `as_of`、延迟、来源记录、数据版本、方法论与警告（§0.2 决策 4）。
  - P0 只读；原子工具默认不嵌套 LLM；不提供任意 SQL / 任意 URL 抓取（§0.2 决策 5）。
  - Postgres 为业务与数据主真值源；DO 用于协调，Workflows/Queues 用于长任务（§0.2 决策 6）。
- **Recommended Defaults:** 首发延迟/EOD；繁中主体验；pgvector 首发检索；加权 credits 计费；MVP 不做交易与个性化建议（PRD §21、§22）。
- **Freedoms:** 模型供应商可替换（业务逻辑不绑定单一 LLM）；套餐定价为验证价可调；UI 布局与模板可迭代。

### Feasibility Boundary

- **Confirmed:** 技术基线 AI SDK v7 + Hono on Cloudflare Workers（不采用 fastclaw）；数据合作方拥有 30+ 年港股数据；既定 Web 栈 TanStack Start + Vite + Hono（PRD §1.1、§22）。
- **[UNKNOWN]:** 数据合作方逐字段再分发权范围；具体功能的 Type 4 判定；30 年历史对外批量/存储限制（PRD §14、§21）。
- **[UNVERIFIED]:** 本仓库为 greenfield（无应用代码），PRD §12 性能目标需在真实数据链路核验后调整。

## Users

### Primary Users

- **自主研究型个人投资者**
  - Need: 快速理解公司、估值、周期与事件，统一口径
  - Success signal: 数分钟内完成一次带证据的研究并保存/比较/继续
- **AI Power User / 开发者**
  - Need: 在 Claude/ChatGPT/IDE/自建 Agent 中调可靠港股工具
  - Success signal: MCP 首次成功调用中位时间 < 10 分钟，结构化输出可直接计算
- **港股新手**
  - Need: 看懂财务与股价变化、术语解释
  - Success signal: 新手模式可读且不降低数据准确性

### Secondary Users

- **资深投资者 / 内容创作者 / 研究团队**
  - Need: 跨公司跨周期批量筛选、可验证图表、团队协作
  - Success signal: 深度历史 + 筛选 + 事件研究 + 可分享证据链
- **数据/合规/运营人员（内部）**
  - Need: 管理授权、质量、成本与事故
  - Success signal: 字段授权可配置、可用 request_id 调查、可对账

## Success Criteria

| Metric | Target | Measurement Method | Degradation Threshold |
|---|---:|---|---:|
| 首次研究完成率 | 70% | 注册后首个研究 run 成功结束占比（§16.5） | 55% |
| 注册 24h 激活率 | 35% | 24h 内保存/观察/继续/MCP 调用占比 | 25% |
| MCP 首次成功调用中位时间 | < 10 min | Developer Console 连接→首调耗时中位（MCP-09） | 20 min |
| 核心工具成功率 | > 99.5% | 排除用户输入/授权错误后的工具成功率（§12.1） | 99.0% |
| 无来源具体金融数字比例 | < 0.1% | 评估集抽样答案中未绑定证据的数字占比（§16.5、AGT-05） | 0.5% |
| Pro/Developer 贡献毛利 | 为正 | 单用户月度贡献毛利（§15.5） | 0 |

## Acceptance Scenarios

### Scenario 1（Web 证据优先研究）

- **Given:** 已授权延迟/EOD 数据的 Pro 用户问“某港股最近为什么下跌”
- **When:** Agent 在 ≤6–8 步内调用注册的只读工具并生成分层答案
- **Then:** 答案区分事实/计算/推断/未知，每个具体数字可点开证据（`source_record_id`+`data_version`），歧义先澄清不静默猜测
- **Machine-checkable evidence:** 评估集中该类答案数字 100% 绑定工具结果或确定性计算；歧义用例返回候选项（SEC-03/AGT-02/AGT-05/AGT-06/AGT-07）

### Scenario 2（MCP 结构化再分发）

- **Given:** 外部 AI 客户端经 OAuth 取得 `market.read`/`fundamentals.read` scope
- **When:** 调用 `get_price_history` 拉取多年历史
- **Then:** 返回 `structuredContent` 含日期/币种/复权/时区/游标，遵守套餐总量与字段授权，附 usage 与 request_id；越权返回 `DATA_NOT_LICENSED`/`SCOPE_DENIED`
- **Machine-checkable evidence:** Schema 校验通过；分页不可绕过总量；权限不足返回标准错误码（MCP-01/04/06/07/08、US-M02/M04/M07）

### Non-goals

- 交易下单/自动交易/代客执行；基于风险承受度的个性化买卖与仓位建议（§4.2、§14.2）
- 全球市场/期权/期货/窝轮/牛熊证/高频逐笔；任意 SQL / 任意 URL / 全市场无限导出（§4.2、§9.4）
- 社区荐股/排行榜/跟单；宣称预测股价或保证收益（§4.2）
